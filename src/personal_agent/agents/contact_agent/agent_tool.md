# 联系人管理工具

## 功能说明
本工具用于管理联系人信息，支持以下功能：
- 添加新联系人
- 查询联系人（按姓名、关系类型）
- 更新联系人信息
- 删除联系人
- 列出所有联系人
- 按关系类型筛选联系人

## 参数说明

### list（列出联系人）
- `relationship`: 关系类型
  - 类型：string
  - 示例："同学"、"同事"、"家人"
  - 说明：可选参数，用于筛选特定关系的联系人

### add（添加联系人）
- `name`: 姓名
  - 类型：string
  - 示例："张三"
  - 说明：必需参数

- `phone`: 电话
  - 类型：string
  - 示例："13800138000"
  - 说明：可选参数

- `email`: 邮箱
  - 类型：string
  - 示例："zhangsan@example.com"
  - 说明：可选参数

- `relationship`: 关系类型
  - 类型：string
  - 示例："同事"、"同学"、"家人"
  - 说明：可选参数

- `address`: 地址
  - 类型：string
  - 示例："北京市朝阳区"
  - 说明：可选参数

- `company`: 公司
  - 类型：string
  - 示例："某某公司"
  - 说明：可选参数

### query（查询联系人）
- `name`: 姓名或关键词
  - 类型：string
  - 示例："张三"、"李四"
  - 说明：必需参数

- `relationship`: 关系类型
  - 类型：string
  - 示例："同事"
  - 说明：可选参数，用于缩小查询范围

### update（更新联系人）
- `name`: 原姓名
  - 类型：string
  - 示例："张三"
  - 说明：必需参数，用于定位联系人

- `new_name`: 新姓名
  - 类型：string
  - 示例："张小三"
  - 说明：可选参数

- `phone`: 新电话
  - 类型：string
  - 示例："13900139000"
  - 说明：可选参数

- `email`: 新邮箱
  - 类型：string
  - 示例："new@example.com"
  - 说明：可选参数

- `relationship`: 新关系类型
  - 类型：string
  - 示例："同事"
  - 说明：可选参数

### delete（删除联系人）
- `name`: 姓名
  - 类型：string
  - 示例："张三"
  - 说明：必需参数

## 使用场景

### 场景1：列出所有联系人
```
用户："列出所有联系人"
调用：list()
```

### 场景2：按关系类型列出联系人
```
用户："同学有哪些"
调用：list(relationship="同学")
```

### 场景3：添加联系人
```
用户："添加联系人张三，电话是13800138000"
调用：add(name="张三", phone="13800138000")
```

### 场景4：查询联系人
```
用户："李四的邮箱"
调用：query(name="李四")
```

### 场景5：更新联系人
```
用户："把张三的电话改成13900139000"
调用：update(name="张三", phone="13900139000")
```

### 场景6：删除联系人
```
用户："删除张三"
调用：delete(name="张三")
```

## 注意事项

### 1. 联系人唯一性
- 联系人通过姓名唯一标识
- 如果姓名重复，会列出所有匹配项供用户选择

### 2. 信息完整性
- 添加联系人时，至少需要提供姓名
- 其他信息（电话、邮箱）可选，但建议尽量完善

### 3. 查询结果
- 查询结果为空时，会提示用户是否添加新联系人
- 查询结果为多个时，会列出所有匹配项供用户选择

### 4. 删除确认
- 删除联系人前会确认用户意图
- 避免误删重要联系人

### 5. 关系类型
- 常用关系类型：家人、同事、同学、朋友、客户
- 可以自定义关系类型

## 常见问题

### Q1：如何查找某个关系的所有联系人？
A：使用list并指定relationship参数：
```
list(relationship="同事")
```

### Q2：联系人姓名重复怎么办？
A：系统会列出所有匹配项，用户可以选择具体要操作哪一个。

### Q3：如何修改联系人的部分信息？
A：使用update，只提供需要修改的字段即可：
```
update(name="张三", phone="13900139000")
```

### Q4：联系人数据存储在哪里？
A：所有联系人数据存储在本地，保护用户隐私。
